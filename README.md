📚 Book Management System (Mini Test)
개발자: 김연주
개발 기간: 2025.12.24
프로젝트 성격: Spring Boot와 JPA를 활용한 도서 정보 관리 시스템 (CRUD 및 파일 업로드 구현)

🛠 기술 스택 (Tech Stack)
Backend
Java 17 
Spring Boot 3.x 
Spring Data JPA: 데이터베이스 연동 및 ORM 처리 
Thymeleaf: 서버 사이드 템플릿 엔진 
Lombok: 보일러플레이트 코드 최적화 (@Data, @AllArgsConstructor 등) 
Oracle Database / H2: 데이터 저장 및 시퀀스 관리 
Frontend
HTML5 / CSS3 
Bootstrap 3.4.1: 반응형 레이아웃 및 UI 구성 
Font Awesome: 아이콘 라이브러리 

🌟 주요 기능 (Key Features)
1. 도서 정보 관리 (CRUD)

입력(Create): 도서명, 작가명, 도서 소개글 및 도서 이미지 업로드 
조회(Read): 전체 도서 목록 출력 및 상세 페이지 확인 
수정(Update): 기존 도서 정보 및 이미지 변경 (새 이미지 미선택 시 기존 이미지 유지 기능 포함) 
삭제(Delete): DB 데이터 삭제와 동시에 로컬 서버에 저장된 이미지 파일 물리적 삭제 처리 

2. 부가 기능

이미지 파일 업로드: UUID를 활용한 파일명 중복 방지 및 지정된 경로 저장 
검색 기능: 도서명 또는 작가명 기반의 동적 키워드 검색 (Native Query 활용) 
조회수 카운트: 상세 페이지 접속 시마다 조회수(readcnt) 증가 로직 구현

[프로젝트 경험 기술서]
프로젝트명: 도서관 도서 관리 및 대여 웹 서비스 (Library Management System)
사용 기술: Java, JSP, Thymeleaf, Apache Tomcat, Oracle DB(또는 MySQL), MyBatis, HTML/CSS

1. 프로젝트 개요
- 목표: 도서 검색, 등록, 수정, 삭제(CRUD+S)가 가능한 웹 기반의 도서관 관리 시스템 구축
- 진행 방식: 개인 미니 프로젝트로 기능 구현 후, 3일간의 집중 팀 프로젝트를 통해 기능 고도화 및 협업 수행
- 핵심 가치: DB와 웹 서버 간의 데이터 바인딩을 이해하고, 파일 업로드 및 리소스 관리 로직을 직접 구현

1. 기술적 구조 및 핵심 구현 (Hard Skills)
이 프로젝트는 데이터베이스 연동과 리소스 관리에 중점을 두었습니다.
- MyBatis 기반의 DB 연동 및 CRUD 구현
    - SQL 쿼리를 코드와 분리하여 관리하는 MyBatis를 도입, 유지보수성을 높임.
    - 사용자 입력 변수를 매핑하여 동적인 쿼리 처리를 수행하고, 웹 페이지에 실시간으로 데이터 출력.
- 파일 업로드 및 이미지 관리 로직 (문제 해결)
    - 문제: DB에 이미지 파일 자체를 저장할 때 발생하는 용량 및 성능 이슈, 그리고 수업 과정 외의 기술적 난관 봉착.
    - 해결: 이미지는 웹 서버의 특정 디렉토리에 저장하고, DB에는 해당 파일의 **'경로(Path)'와 '파일명'**만 저장하는 방식으로 구조를 설계하여 해결.
- 데이터 무결성 및 서버 리소스 최적화
    - 이미지 수정 로직: 단순 덮어쓰기가 아니라, 기존 이미지를 서버 디렉토리에서 물리적으로 삭제(File.delete())한 후 새 이미지를 저장하는 로직을 구현하여 불필요한 서버 용량 낭비를 방지함.

1. 협업 및 성장 과정 (Soft Skills)
- 능동적 기술 학습: 수업 커리큘럼에 없는 '이미지 수정/삭제 로직'을 구현하기 위해 공식 문서와 기술 블로그를 탐색하며 스스로 해답을 찾아냄.
- 팀워크 및 커뮤니케이션: 3일이라는 짧은 기간 동안 팀원들과 아이디어 회의를 주도하며 구현 가능한 기능(Option)의 범위를 조율하고, 각자의 역할(프론트/백엔드/DB)을 명확히 하여 프로젝트를 완수함.
